# 1.5 分类问题的细分：分类、序列标注与生成

## 1.5.1 从输入输出结构的视角重新审视分类

在上一节中，我们讨论了分类作为一种基本任务类型。但当处理序列数据（如文本、语音、时间序列）时，"分类"这一概念呈现出更丰富的形态。根据输入与输出之间的结构关系，可以将序列上的预测任务细分为三种形式：

1. **序列分类（Sequence Classification）**：整个序列映射到单一标签
2. **序列标注（Sequence Labeling）**：序列中每个元素映射到一个标签
3. **序列生成（Sequence Generation）**：序列映射到另一个（可能不等长的）序列

这三种形式在自然语言处理、语音识别、生物信息学等领域有着广泛应用。理解它们的数学本质和建模方法，是掌握现代序列模型（如 Transformer、大语言模型）的重要前提。

## 1.5.2 序列分类：多对一映射

### 问题定义

序列分类（Sequence Classification）的目标是将一个变长序列 $\mathbf{x} = (x_1, x_2, ..., x_T)$ 映射到一个类别标签 $y \in \{1, 2, ..., K\}$。举个例子，你读完一篇电影评论，判断它是"好评"还是"差评"——你需要综合整篇文字的信息，最终给出一个单一的判断。这就是序列分类：多个输入元素压缩为一个输出。

$$f: \mathcal{X}^* \rightarrow \{1, 2, ..., K\}$$

其中 $\mathcal{X}^*$ 表示由 $\mathcal{X}$ 中元素组成的任意长度序列的集合。

这里的关键挑战是：如何将变长的输入序列压缩为固定长度的表示，以便输入到分类器中。

### 典型应用

**文本分类**：判断一段文本的类别
- 情感分析：电影评论 → {正面, 负面, 中性}
- 主题分类：新闻文章 → {政治, 体育, 科技, ...}
- 意图识别：用户query → {查询天气, 播放音乐, ...}

**文档级任务**：
- 垃圾邮件检测：邮件内容 → {垃圾邮件, 正常邮件}
- 作者归属：文章 → {作者A, 作者B, ...}

**语音分类**：
- 语音命令识别：音频片段 → {上, 下, 左, 右, ...}
- 说话人识别：音频 → {说话人1, 说话人2, ...}

### 建模方法

**基于 RNN 的方法**：

循环神经网络（RNN）及其变体（LSTM、GRU）逐步处理序列，维护一个隐状态。这就像你逐字阅读一篇文章，读每个新字时都会更新你对全文的理解状态：

$$h_t = f(h_{t-1}, x_t)$$

最终的隐状态 $h_T$ 相当于你读完整篇文章后的“综合印象”，用它来做分类决策：

$$P(y|\mathbf{x}) = \text{softmax}(W h_T + b)$$

LSTM 通过门控机制缓解了梯度消失问题，使其能够捕获长距离依赖：

$$\begin{aligned}
f_t &= \sigma(W_f [h_{t-1}, x_t] + b_f) & \text{(遗忘门)} \\
i_t &= \sigma(W_i [h_{t-1}, x_t] + b_i) & \text{(输入门)} \\
\tilde{c}_t &= \tanh(W_c [h_{t-1}, x_t] + b_c) & \text{(候选记忆)} \\
c_t &= f_t \odot c_{t-1} + i_t \odot \tilde{c}_t & \text{(记忆更新)} \\
o_t &= \sigma(W_o [h_{t-1}, x_t] + b_o) & \text{(输出门)} \\
h_t &= o_t \odot \tanh(c_t) & \text{(隐状态)}
\end{aligned}$$

**基于 CNN 的方法**：

卷积神经网络通过滑动窗口捕获局部 n-gram 特征，然后通过池化操作聚合为全局表示。TextCNN 使用多种尺寸的卷积核捕获不同粒度的模式：

$$h_i^{(k)} = \text{ReLU}(W^{(k)} x_{i:i+k-1} + b^{(k)})$$

最大池化选择每个特征图的最显著特征：

$$\hat{h}^{(k)} = \max_i h_i^{(k)}$$

**基于 Transformer 的方法**：

Transformer 通过自注意力机制建模序列中任意两个位置之间的依赖：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

对于序列分类，BERT 等预训练模型引入特殊的 [CLS] 标记，其最终表示用于分类：

$$P(y|\mathbf{x}) = \text{softmax}(W h_{[CLS]} + b)$$

### 序列表示的聚合策略

将变长序列压缩为固定表示的常见方法：

- **最后隐状态**：$h_T$（适用于 RNN）
- **平均池化**：$\frac{1}{T}\sum_{t=1}^T h_t$
- **最大池化**：$\max_t h_t$（元素级）
- **注意力池化**：$\sum_t \alpha_t h_t$，其中 $\alpha_t = \text{softmax}(w^T h_t)$
- **特殊标记**：[CLS] 表示（BERT 风格）

## 1.5.3 序列标注：多对多等长映射

### 问题定义

序列标注（Sequence Labeling）的目标是为输入序列的每个元素分配一个标签。给定输入序列 $\mathbf{x} = (x_1, ..., x_T)$，输出等长的标签序列 $\mathbf{y} = (y_1, ..., y_T)$，其中 $y_t \in \{1, 2, ..., K\}$。

$$f: \mathcal{X}^T \rightarrow \mathcal{Y}^T$$

序列标注与序列分类的关键区别在于：序列分类只需要一个全局决策，而序列标注需要对每个位置做出决策，且相邻位置的决策可能相互依赖。

### 典型应用

**命名实体识别（Named Entity Recognition, NER）**：

识别文本中的人名、地名、组织名等实体。

输入：`李明 在 北京大学 学习 计算机`

输出：`B-PER O B-ORG I-ORG I-ORG O O`

BIO 标注体系：
- B-X：实体 X 的开始
- I-X：实体 X 的延续
- O：非实体

**词性标注（Part-of-Speech Tagging）**：

为每个词标注词性。

输入：`The quick brown fox jumps`

输出：`DT JJ JJ NN VBZ`

**中文分词**：

判断每个字符是词的开始、中间还是结尾。

输入：`中华人民共和国`

输出：`B M M E B M E`（中华人民 | 共和国）

**语音识别的帧级标注**：

为每个音频帧标注音素。

### 标签依赖性建模

序列标注的一个核心问题是相邻标签之间的依赖关系。例如，在 NER 中，B-PER 后面只能跟 I-PER 或 O，不能跟 I-ORG。简单地对每个位置独立分类会忽略这种约束。这就好比你在组装一套家具：每个零件的位置不是独立的，螺丝必须在螺母旁边，桌腓必须在桌面下方。忽视这些约束只会产生不合理的结果。

**条件随机场（Conditional Random Field, CRF）**

CRF 是处理标签依赖的经典方法。线性链 CRF 将标签序列的条件概率建模为：

$$P(\mathbf{y}|\mathbf{x}) = \frac{1}{Z(\mathbf{x})} \exp\left(\sum_{t=1}^T \phi(y_{t-1}, y_t, \mathbf{x}, t)\right)$$

其中 $Z(\mathbf{x})$ 是配分函数（归一化常数），$\phi$ 是特征函数。

在神经网络框架下，CRF 层通常只建模标签之间的转移：

$$P(\mathbf{y}|\mathbf{x}) = \frac{1}{Z(\mathbf{x})} \exp\left(\sum_{t=1}^T (E_{y_t}(x_t) + T_{y_{t-1}, y_t})\right)$$

其中 $E_{y_t}(x_t)$ 是神经网络输出的发射分数，$T$ 是可学习的转移矩阵。

**训练**：最大化正确标签序列的对数似然

$$\log P(\mathbf{y}|\mathbf{x}) = \sum_t (E_{y_t} + T_{y_{t-1}, y_t}) - \log Z(\mathbf{x})$$

配分函数 $Z$ 通过前向算法在 $O(TK^2)$ 时间内计算。

**解码**：使用 Viterbi 算法找到最可能的标签序列

$$\mathbf{y}^* = \arg\max_\mathbf{y} P(\mathbf{y}|\mathbf{x})$$

**BiLSTM-CRF 架构**

经典的序列标注架构结合了双向 LSTM 的表示能力和 CRF 的标签依赖建模：

1. **嵌入层**：将输入 token 转换为向量
2. **BiLSTM 层**：双向编码上下文信息
3. **CRF 层**：建模标签转移，输出最优标签序列

$$h_t = [\overrightarrow{h}_t; \overleftarrow{h}_t]$$

$$E_k(x_t) = W_k^T h_t + b_k$$

### 评估指标

序列标注的评估通常在实体级别（而非 token 级别）进行：

**实体级 F1**：只有当实体的边界和类型都完全正确时，才算正确。

$$\text{Precision} = \frac{\text{正确预测的实体数}}{\text{预测的实体总数}}$$

$$\text{Recall} = \frac{\text{正确预测的实体数}}{\text{真实的实体总数}}$$

这比 token 级准确率更严格，也更能反映实际应用的效果。

## 1.5.4 序列生成：多对多变长映射

### 问题定义

序列生成（Sequence Generation）的目标是将输入序列映射到一个可能不等长的输出序列。给定输入 $\mathbf{x} = (x_1, ..., x_S)$，生成输出 $\mathbf{y} = (y_1, ..., y_T)$，其中 $S$ 和 $T$ 可以不同。

$$f: \mathcal{X}^* \rightarrow \mathcal{Y}^*$$

序列生成是最复杂的序列任务形式：输出长度可变，且每个输出位置依赖于之前的输出。

### 典型应用

**机器翻译（Machine Translation）**：

输入：`The weather is nice today.`

输出：`今天天气很好。`

源语言和目标语言的句子长度通常不同，词序也可能变化。

**文本摘要（Text Summarization）**：

输入：一篇长文档

输出：一段简短的摘要

**对话系统（Dialogue System）**：

输入：用户的问题或陈述

输出：系统的回复

**代码生成（Code Generation）**：

输入：自然语言描述

输出：对应的代码

### Encoder-Decoder 架构

序列生成的标准框架是 Encoder-Decoder（编码器-解码器）架构：

**编码器**：将输入序列压缩为固定长度的上下文向量或一组上下文向量。

$$\mathbf{h} = \text{Encoder}(\mathbf{x}) = (h_1, ..., h_S)$$

**解码器**：基于上下文向量自回归地生成输出序列。

$$P(\mathbf{y}|\mathbf{x}) = \prod_{t=1}^T P(y_t | y_1, ..., y_{t-1}, \mathbf{h})$$

在每个时间步，解码器输出下一个 token 的概率分布：

$$P(y_t | y_{<t}, \mathbf{h}) = \text{softmax}(W s_t + b)$$

其中 $s_t$ 是解码器的隐状态。

### 注意力机制

原始的 Encoder-Decoder 架构将整个输入压缩为一个固定向量，这在处理长序列时成为瓶颈（信息瓶颈问题）。注意力机制（Attention Mechanism）允许解码器在每一步动态地关注输入的不同部分。这就像翻译时你不会先把整篇文章暂存在脑子里再动笔，而是翻到每个句子时回头看对应的原文部分。

**计算过程**：

1. 计算解码器当前状态 $s_{t-1}$ 与每个编码器隐状态 $h_i$ 的相关性分数：

$$e_{ti} = a(s_{t-1}, h_i)$$

常用的评分函数包括：
- 点积：$e_{ti} = s_{t-1}^T h_i$
- 双线性：$e_{ti} = s_{t-1}^T W h_i$
- 加性：$e_{ti} = v^T \tanh(W_1 s_{t-1} + W_2 h_i)$

2. 归一化得到注意力权重：

$$\alpha_{ti} = \frac{\exp(e_{ti})}{\sum_j \exp(e_{tj})}$$

3. 计算上下文向量（编码器隐状态的加权和）：

$$c_t = \sum_i \alpha_{ti} h_i$$

4. 结合上下文向量更新解码器状态：

$$s_t = f(s_{t-1}, y_{t-1}, c_t)$$

### Transformer 与自回归生成

Transformer 架构抛弃了循环结构，完全基于注意力机制。其自注意力（Self-Attention）允许每个位置直接关注序列中的所有其他位置。

**编码器**：由多层自注意力和前馈网络组成，双向建模输入。

**解码器**：在自注意力基础上增加掩码（Masked Self-Attention），确保位置 $t$ 只能看到 $t$ 之前的位置（保证自回归性）。同时增加交叉注意力（Cross-Attention）关注编码器输出。

$$\text{MaskedAttention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + M\right)V$$

其中 $M$ 是掩码矩阵，未来位置的值为 $-\infty$。

### 大语言模型：纯解码器架构

GPT 系列模型采用纯解码器（Decoder-only）架构，将所有任务统一为序列生成：

$$P(\mathbf{y}|\mathbf{x}) = \prod_{t=1}^T P(y_t | x_1, ..., x_S, y_1, ..., y_{t-1})$$

输入和输出被拼接成一个序列，模型被训练为预测下一个 token。这种统一的框架使得大语言模型能够处理各种任务——分类变成生成类别名称，问答变成生成答案文本。

### 解码策略

训练时使用 teacher forcing（用真实的前缀训练预测下一个 token），但推理时需要使用模型自己的预测。常用的解码策略包括：

**贪婪解码（Greedy Decoding）**：

每一步选择概率最大的 token：

$$y_t = \arg\max_y P(y|y_{<t}, \mathbf{x})$$

简单高效，但容易陷入局部最优。

**束搜索（Beam Search）**：

维护 $B$ 个最佳候选序列，每一步扩展所有候选并保留得分最高的 $B$ 个：

$$\text{score}(\mathbf{y}_{1:t}) = \sum_{\tau=1}^t \log P(y_\tau | y_{<\tau}, \mathbf{x})$$

$B$ 称为束宽（Beam Width），$B=1$ 退化为贪婪解码。

**采样方法**：

从概率分布中随机采样，增加生成的多样性。

**温度采样**：用温度参数 $\tau$ 调整分布的平滑度

$$P'(y) = \frac{\exp(\log P(y) / \tau)}{\sum_{y'} \exp(\log P(y') / \tau)}$$

$\tau < 1$ 使分布更尖锐（更确定性），$\tau > 1$ 使分布更平滑（更随机）。你可以把温度类比为作家的"创作自由度"——低温度时每个词都选最可预测的，像新闻稿一样中规中矩；高温度时会出现更多意外的组合，像诗歌一样充满惊喜（也可能充满胡言乱语）。

**Top-k 采样**：只从概率最高的 $k$ 个 token 中采样

**Top-p（Nucleus）采样**：从累积概率达到 $p$ 的最小 token 集合中采样

### 序列生成的评估

**BLEU（Bilingual Evaluation Understudy）**：

基于 n-gram 精确率的指标，最初用于机器翻译评估：

$$\text{BLEU} = \text{BP} \cdot \exp\left(\sum_{n=1}^N w_n \log p_n\right)$$

其中 $p_n$ 是 n-gram 精确率，BP 是长度惩罚因子。

BLEU 的局限性在于只考虑精确匹配，无法捕捉语义相似但用词不同的情况。

**ROUGE（Recall-Oriented Understudy for Gisting Evaluation）**：

基于召回率的指标，常用于文本摘要评估。ROUGE-L 基于最长公共子序列。

**困惑度（Perplexity）**：

语言模型的内在评估指标：

$$\text{PPL} = \exp\left(-\frac{1}{T}\sum_{t=1}^T \log P(y_t|y_{<t})\right)$$

困惑度越低，模型对数据的拟合越好。

**人工评估**：

对于开放式生成任务，自动指标与人类判断的相关性有限。人工评估通常考虑流畅性、相关性、事实正确性等维度。

## 1.5.5 三种形式的统一视角

从建模的角度，三种任务形式可以在统一框架下理解：

| 任务类型 | 输入 | 输出 | 条件独立性假设 |
|---------|------|------|---------------|
| 序列分类 | 序列 $(x_1, ..., x_T)$ | 单一标签 $y$ | $y$ 依赖于整个序列 |
| 序列标注 | 序列 $(x_1, ..., x_T)$ | 序列 $(y_1, ..., y_T)$ | $y_t$ 可能依赖于 $y_{t-1}$ |
| 序列生成 | 序列 $(x_1, ..., x_S)$ | 序列 $(y_1, ..., y_T)$ | $y_t$ 依赖于 $y_{<t}$ |

**Transformer 架构的统一**：

Transformer 的灵活性使其能够处理所有三种任务：

- **序列分类**：使用 [CLS] 表示或池化后的表示进行分类（BERT）
- **序列标注**：对每个位置的表示进行分类（BERT + 分类头）
- **序列生成**：自回归生成或 Encoder-Decoder 生成（GPT、T5）

**大语言模型的任务统一**：

GPT 等大语言模型将所有任务统一为生成：

- 分类：生成类别名称（"这段评论的情感是：正面"）
- 序列标注：生成标注后的序列（"北京大学 → [ORG]北京大学"）
- 生成：直接生成目标序列

这种统一简化了模型设计，也是大模型涌现能力的基础。
