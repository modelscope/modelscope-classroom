# 2.1 深度学习领域的基本概念

## 2.1.1 模型与假设空间

在机器学习的语境中，**模型**（Model）是一个参数化的函数族。选择一个模型，就是选择了一个假设空间——所有可能的函数构成的集合。

考虑线性回归模型：

$$f(x; w, b) = w^T x + b$$

这个模型定义了一个假设空间：所有形如 $w^T x + b$ 的仿射函数。不同的参数 $(w, b)$ 对应不同的具体函数，但它们都属于同一个假设空间。无论我们如何调整参数，线性回归模型永远无法表示 $y = x^2$ 这样的非线性函数——这不在它的假设空间内。

深度神经网络的假设空间要丰富得多。一个 $L$ 层的前馈网络可以表示为：

$$f(x; \theta) = f_L \circ f_{L-1} \circ \cdots \circ f_1(x)$$

其中每一层 $f_l(h) = \sigma(W_l h + b_l)$，$\sigma$ 是非线性激活函数。参数集合 $\theta = \{W_1, b_1, ..., W_L, b_L\}$ 可能包含数百万甚至数十亿个标量。根据通用近似定理（Universal Approximation Theorem），只要网络足够宽或足够深，神经网络可以以任意精度近似任何连续函数。

**模型容量**（Model Capacity）描述了模型能够表示的函数的丰富程度。高容量的模型可以拟合更复杂的数据模式，但也更容易过拟合。模型容量是一个定性概念；其量化形式包括 VC 维、Rademacher 复杂度等。

## 2.1.2 参数与超参数

机器学习中的"参数"一词有两种不同的含义，需要仔细区分。

**参数**（Parameters）是模型内部的可学习变量，通过训练过程从数据中自动确定。神经网络的权重矩阵 $W$ 和偏置向量 $b$ 是典型的参数。参数的数量决定了模型的规模——GPT-3 有 1750 亿参数，LLaMA-70B 有 700 亿参数。

**超参数**（Hyperparameters）是在训练开始之前由人工设定的配置，控制着学习过程本身。常见的超参数包括：

| 类别 | 超参数 | 作用 |
|------|--------|------|
| 模型结构 | 层数、隐藏单元数、激活函数 | 定义假设空间 |
| 优化 | 学习率、batch size、优化器类型 | 控制参数更新方式 |
| 正则化 | dropout 率、weight decay、early stopping | 防止过拟合 |
| 训练流程 | epoch 数、warmup 步数 | 控制训练时长 |

超参数的选择对模型性能有重大影响，但没有通用的最优值——最佳超参数取决于具体任务、数据集和计算资源。超参数调优（Hyperparameter Tuning）是机器学习工程的重要环节，常用方法包括网格搜索、随机搜索和贝叶斯优化。

## 2.1.3 前向传播

**前向传播**（Forward Propagation）是从输入到输出的计算过程。给定输入 $x$，模型按层顺序计算，最终产生预测 $\hat{y}$。

以一个两层神经网络为例：

$$h = \sigma(W_1 x + b_1) \quad \text{(隐藏层)}$$
$$\hat{y} = W_2 h + b_2 \quad \text{(输出层)}$$

前向传播的计算图可以表示为：

```
x → [Linear] → [Activation] → h → [Linear] → ŷ
     W₁, b₁        σ                W₂, b₂
```

前向传播是确定性的：给定相同的输入和参数，输出总是相同的。在训练阶段，前向传播的结果用于计算损失；在推理阶段，前向传播直接给出预测。

现代深度学习框架（PyTorch、TensorFlow）会在前向传播过程中自动构建**计算图**（Computational Graph），记录每一步操作及其输入输出。这个计算图是反向传播的基础。

## 2.1.4 损失函数

**损失函数**（Loss Function）量化了模型预测与真实标签之间的差距。损失越小，预测越准确。

对于单个样本 $(x, y)$，损失函数 $L(y, \hat{y})$ 接受真实标签 $y$ 和预测值 $\hat{y}$ 作为输入，输出一个非负实数。对于整个训练集，我们通常计算平均损失：

$$\mathcal{L}(\theta) = \frac{1}{n} \sum_{i=1}^n L(y_i, f(x_i; \theta))$$

损失函数的选择取决于任务类型：

**回归任务**：
- 均方误差（MSE）：$L(y, \hat{y}) = (y - \hat{y})^2$
- 平均绝对误差（MAE）：$L(y, \hat{y}) = |y - \hat{y}|$

**分类任务**：
- 交叉熵损失：$L(y, \hat{p}) = -\sum_k y_k \log \hat{p}_k$

损失函数设计有两个核心考量：一是损失应当准确反映任务目标（对于分类，我们关心分类是否正确）；二是损失应当便于优化（光滑、可微、最好是凸的）。这两个目标有时会冲突——0-1 损失最直接地反映分类准确率，但它不可微，无法用梯度下降优化。交叉熵是 0-1 损失的一个光滑替代，既保留了分类的语义，又便于优化。

## 2.1.5 反向传播

**反向传播**（Backpropagation）是计算损失函数对每个参数梯度的算法。它是训练神经网络的核心技术。

数学上，反向传播是链式法则（Chain Rule）的系统应用。设损失为 $\mathcal{L}$，我们要计算 $\frac{\partial \mathcal{L}}{\partial W_l}$ 对于每一层的权重 $W_l$。

考虑一个简单的计算链：

$$x \xrightarrow{W_1} h_1 \xrightarrow{W_2} h_2 \xrightarrow{} \mathcal{L}$$

根据链式法则：

$$\frac{\partial \mathcal{L}}{\partial W_1} = \frac{\partial \mathcal{L}}{\partial h_2} \cdot \frac{\partial h_2}{\partial h_1} \cdot \frac{\partial h_1}{\partial W_1}$$

反向传播从输出层开始，逐层向输入方向计算梯度。每一层接收来自后一层的梯度 $\frac{\partial \mathcal{L}}{\partial h_{l+1}}$，计算本层的梯度 $\frac{\partial \mathcal{L}}{\partial h_l}$ 和参数梯度 $\frac{\partial \mathcal{L}}{\partial W_l}$，然后将 $\frac{\partial \mathcal{L}}{\partial h_l}$ 传递给前一层。

反向传播的计算复杂度与前向传播相当（同阶），但需要存储前向传播中间结果（用于计算局部梯度），因此内存开销较大。这是深度学习训练中 GPU 显存成为瓶颈的主要原因之一。

**自动微分**（Automatic Differentiation）是现代深度学习框架的核心功能。程序员只需定义前向计算，框架自动构建计算图并实现反向传播。PyTorch 的 `loss.backward()` 一行代码就完成了整个反向传播过程。

## 2.1.6 训练循环

将以上概念串联起来，神经网络的训练过程可以概括为一个迭代循环：

```python
for epoch in range(num_epochs):
    for batch in dataloader:
        # 1. Forward propagation
        predictions = model(batch.inputs)
        
        # 2. Compute loss
        loss = loss_function(predictions, batch.labels)
        
        # 3. Backward propagation
        optimizer.zero_grad()  # Clear previous gradients
        loss.backward()        # Compute gradients
        
        # 4. Parameter update
        optimizer.step()       # Update parameters using gradients
```

**Epoch**：遍历整个训练集一次称为一个 epoch。训练通常需要多个 epoch，因为单次遍历不足以让模型收敛。

**Batch**：由于计算资源限制，我们不会一次性处理所有数据，而是将数据划分为小批量（mini-batch）。每个 batch 独立进行前向传播、损失计算、反向传播和参数更新。

**Iteration/Step**：处理一个 batch 称为一个 iteration 或 step。一个 epoch 包含 $\lceil n / \text{batch\_size} \rceil$ 个 iteration。

## 2.1.7 张量与张量运算

深度学习的计算对象是**张量**（Tensor）——多维数组的推广。

| 维度 | 名称 | 示例 |
|------|------|------|
| 0 | 标量 | 损失值 3.14 |
| 1 | 向量 | 词嵌入 [0.1, 0.3, -0.2, ...] |
| 2 | 矩阵 | 权重矩阵 $W \in \mathbb{R}^{m \times n}$ |
| 3 | 3D 张量 | 批量词嵌入 (batch, seq_len, embed_dim) |
| 4 | 4D 张量 | 图像批量 (batch, channel, height, width) |

深度学习框架对张量运算进行了高度优化，充分利用 GPU 的并行计算能力。理解张量的形状（shape）和运算规则是调试神经网络代码的基本功。

**广播**（Broadcasting）是张量运算中的重要机制：形状不完全匹配的张量在运算时会自动扩展。例如，形状为 (3, 4) 的矩阵加上形状为 (4,) 的向量，向量会被广播为 (3, 4)，相当于每一行都加上这个向量。

## 2.1.8 计算图与自动微分

**计算图**（Computational Graph）是将计算过程表示为有向无环图（DAG）的形式。节点代表变量或操作，边代表数据流向。

考虑计算 $z = (x + y) \times y$：

```
x ─┬─→ [+] ─→ temp ─┬─→ [×] ─→ z
   │                │
y ─┴────────────────┘
```

计算图的优势在于：

1. **模块化**：复杂计算可以分解为简单操作的组合
2. **自动求导**：图中每条边都对应一个局部梯度，反向遍历图即可计算所有梯度
3. **优化执行**：框架可以对图进行优化（算子融合、内存复用等）

PyTorch 使用**动态计算图**（Dynamic Computational Graph）：每次前向传播都构建新的图。这使得调试更方便，可以使用标准 Python 控制流。TensorFlow 1.x 使用静态图，需要先定义图再执行，但 TensorFlow 2.x 默认启用 Eager Execution，行为更接近 PyTorch。
