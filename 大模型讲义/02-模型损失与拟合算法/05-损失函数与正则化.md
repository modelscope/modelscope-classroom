# 2.5 损失函数与正则化

## 2.5.1 损失函数的作用

损失函数（Loss Function）是机器学习的核心组件，它定义了"什么是好的预测"。模型训练的本质是最小化损失函数，因此损失函数的设计直接决定了模型学到什么样的知识。就像考试评分规则决定了学生的复习策略：如果只看对错给 0/1 分，学生只需保证答对；如果按偏离程度扣分，学生会力求精确。

好的损失函数应当满足：
1. **任务相关**：损失应当反映实际任务的目标
2. **可优化**：损失应当便于计算梯度和优化
3. **鲁棒**：损失对噪声和异常值有适当的敏感度

本节将系统介绍回归和分类任务中的常用损失函数，并从信息论角度深入理解熵、交叉熵与 KL 散度。

## 2.5.2 回归损失函数

### 均方误差（MSE）

均方误差是最常用的回归损失：

$$L_{MSE}(y, \hat{y}) = (y - \hat{y})^2$$

**性质**：
- 连续可微，便于梯度优化
- 对大误差惩罚更重（平方关系）
- 对应高斯噪声假设下的极大似然估计

**缺点**：对异常值敏感。一个极端的异常值会产生很大的损失，可能主导整个优化过程。想象全班同学考试成绩都在 70-90 分，突然来了一个 800 分的异常数据，平方损失会被这个极端值严重干扰。

### 平均绝对误差（MAE）

$$L_{MAE}(y, \hat{y}) = |y - \hat{y}|$$

**性质**：
- 对异常值更鲁棒（线性惩罚而非平方）
- 对应拉普拉斯噪声假设下的极大似然估计

**缺点**：在零点不可微，优化时需要特殊处理（如次梯度）。在误差较小时，梯度恒为常数（±1），可能导致震荡。

### Huber 损失

Huber 损失结合了 MSE 和 MAE 的优点：

$$L_\delta(y, \hat{y}) = \begin{cases} \frac{1}{2}(y - \hat{y})^2 & |y - \hat{y}| \leq \delta \\ \delta |y - \hat{y}| - \frac{1}{2}\delta^2 & |y - \hat{y}| > \delta \end{cases}$$

当误差小于阈值 $\delta$ 时使用平方损失，当误差大于 $\delta$ 时使用线性损失。这使得 Huber 损失既保持了 MSE 的平滑性，又具有 MAE 对异常值的鲁棒性。

### 分位数损失

分位数损失用于预测分布的特定分位数（而非均值）：

$$L_\tau(y, \hat{y}) = \begin{cases} \tau(y - \hat{y}) & y \geq \hat{y} \\ (1-\tau)(\hat{y} - y) & y < \hat{y} \end{cases}$$

当 $\tau = 0.5$ 时，最小化分位数损失等价于预测中位数。$\tau = 0.1$ 对应第 10 百分位数，$\tau = 0.9$ 对应第 90 百分位数。

## 2.5.3 分类损失函数

### 0-1 损失

最直接的分类损失是 0-1 损失：

$$L_{0-1}(y, \hat{y}) = \mathbb{I}[y \neq \hat{y}]$$

0-1 损失直接计算分类错误次数。但它是不连续的、不可微的，无法用于梯度优化。实践中使用其连续的替代损失。

### 交叉熵损失

交叉熵是分类任务最常用的损失函数。对于二分类：

$$L_{CE}(y, p) = -[y \log p + (1-y) \log(1-p)]$$

其中 $y \in \{0, 1\}$ 是真实标签，$p \in (0, 1)$ 是预测的正类概率。

对于 $K$ 类分类：

$$L_{CE}(y, \mathbf{p}) = -\sum_{k=1}^K y_k \log p_k$$

其中 $y$ 是独热编码的标签向量，$\mathbf{p}$ 是预测的概率分布。

**性质**：
- 是 0-1 损失的凸上界
- 对应多项式分布假设下的极大似然估计（负对数似然）
- 当预测置信度错误时惩罚很大（$\log$ 函数在接近 0 时趋于负无穷）

### Hinge 损失

Hinge 损失是 SVM 使用的损失函数：

$$L_{hinge}(y, f(x)) = \max(0, 1 - y \cdot f(x))$$

其中 $y \in \{-1, +1\}$，$f(x)$ 是模型输出的分数（非概率）。

Hinge 损失只惩罚分类错误或分类正确但置信度不足（$y \cdot f(x) < 1$）的样本。正确分类且置信度高的样本损失为零，这促使模型追求大间隔。

### Focal Loss

Focal Loss 专为类别不平衡问题设计：

$$L_{focal}(y, p) = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

其中 $p_t = p$ 如果 $y = 1$，否则 $p_t = 1 - p$。$\gamma \geq 0$ 是聚焦参数，$\alpha_t$ 是类别平衡权重。

当 $\gamma > 0$ 时，容易分类的样本（$p_t$ 接近 1）的损失被大幅下调，使训练聚焦于困难样本。

### 熵

信息熵（Entropy）衡量随机变量的不确定性：

$$H(X) = -\sum_x P(x) \log P(x) = \mathbb{E}[-\log P(X)]$$

其中 $P(x)$ 表示随机变量 $X$ 取值 $x$ 的概率，$\log$ 通常取自然对数或以 2 为底（对应 bit）。

熵衡量的是“惊喜程度的平均值”——如果一个事件的结果很难预测，每次揭晓都能带来很多信息（高熵）；如果结果几乎确定，揭晓它几乎不带来任何新信息（低熵）。

熵越高，不确定性越大。对于离散均匀分布，熵取最大值 $\log K$（$K$ 个状态）；对于确定性分布（一个状态概率为 1），熵为 0。

换个角度理解：明天的天气如果只有"晴"和"雨"两种可能，且各占 50%，你完全无法预判，不确定性最大（熵 = 1 bit）。但如果晴天概率 99%，你几乎可以确定明天是晴天，不确定性很低（熵接近 0）。

### 交叉熵

交叉熵衡量用分布 $Q$ 编码来自分布 $P$ 的数据时的平均编码长度：

$$H(P, Q) = -\sum_x P(x) \log Q(x) = \mathbb{E}_{x \sim P}[-\log Q(x)]$$

其中 $P$ 表示真实分布（标签），$Q$ 表示模型预测的分布。

用大白话讲：如果你用一套“错误的理解”（$Q$）去编码由真实规律（$P$）产生的数据，你平均每次需要多少“字节”。$Q$ 离 $P$ 越远，浪费越大，交叉熵越高。

在机器学习中，$P$ 是真实分布（标签），$Q$ 是模型预测的分布。最小化交叉熵等价于让 $Q$ 尽可能接近 $P$。

交叉熵总是大于等于真实熵：$H(P, Q) \geq H(P)$，等号当且仅当 $P = Q$ 时成立。

### KL 散度

KL 散度（Kullback-Leibler Divergence）衡量两个分布之间的差异：

$$D_{KL}(P \| Q) = \sum_x P(x) \log \frac{P(x)}{Q(x)} = H(P, Q) - H(P)$$

其中第二个等号揭示了 KL 散度与交叉熵的关系：KL 散度 = 交叉熵 - 真实熵，即用 $Q$ 编码相比于用 $P$ 自己编码多浪费了多少。

**性质**：
- 非负性：$D_{KL}(P \| Q) \geq 0$，等号当且仅当 $P = Q$
- 不对称性：$D_{KL}(P \| Q) \neq D_{KL}(Q \| P)$（这就是为什么它不是"距离"）

在优化中，由于 $H(P)$ 是常数（真实分布固定），最小化交叉熵等价于最小化 KL 散度：

$$\arg\min_Q H(P, Q) = \arg\min_Q D_{KL}(P \| Q)$$

### 交叉熵与极大似然的等价性

设真实数据分布为 $P_{data}$，模型分布为 $P_\theta$。最小化交叉熵：

$$\min_\theta H(P_{data}, P_\theta) = \min_\theta \mathbb{E}_{x \sim P_{data}}[-\log P_\theta(x)]$$

用经验分布近似 $P_{data}$：

$$\approx \min_\theta \frac{1}{n} \sum_{i=1}^n [-\log P_\theta(x_i)] = \max_\theta \frac{1}{n} \sum_{i=1}^n \log P_\theta(x_i)$$

这正是极大似然估计。因此，**交叉熵损失 = 负对数似然 = 极大似然估计**。

## 2.5.5 正则化

正则化（Regularization）在损失函数中添加惩罚项，限制模型复杂度。

### L2 正则化（Ridge / Weight Decay）

$$\mathcal{L}_{reg} = \mathcal{L}_{data} + \frac{\lambda}{2} \|w\|_2^2 = \mathcal{L}_{data} + \frac{\lambda}{2} \sum_i w_i^2$$

L2 正则化使权重倾向于取较小的值，但不会精确为零。

**几何解释**：L2 正则化等价于约束优化问题 $\min_w \mathcal{L}_{data}$ s.t. $\|w\|_2 \leq c$。等高线与 L2 球的切点通常不在坐标轴上，因此解不稀疏。

**贝叶斯解释**：L2 正则化等价于对权重施加零均值高斯先验 $w \sim \mathcal{N}(0, \sigma^2 I)$，其中 $\sigma^2 = 1/\lambda$。

**梯度更新**：

$$w \leftarrow w - \eta \nabla_w \mathcal{L}_{data} - \eta \lambda w = (1 - \eta\lambda) w - \eta \nabla_w \mathcal{L}_{data}$$

每步更新都将权重乘以 $(1 - \eta\lambda) < 1$，这就是"权重衰减"名称的由来。

### L1 正则化（LASSO）

$$\mathcal{L}_{reg} = \mathcal{L}_{data} + \lambda \|w\|_1 = \mathcal{L}_{data} + \lambda \sum_i |w_i|$$

L1 正则化诱导稀疏解——许多权重恰好为零。

**几何解释**：L1 约束区域是超立方体，其角点在坐标轴上。等高线与超立方体的切点倾向于落在角点（稀疏解）。

**贝叶斯解释**：L1 正则化等价于拉普拉斯先验。

**稀疏性的直觉**：$|w|$ 的梯度在零点附近是常数（±1），这产生了将权重推向零的恒定压力。而 $w^2$ 的梯度在零点附近趋于零，推动力减弱。

### 弹性网（Elastic Net）

弹性网结合 L1 和 L2：

$$\mathcal{L}_{reg} = \mathcal{L}_{data} + \lambda_1 \|w\|_1 + \lambda_2 \|w\|_2^2$$

当特征高度相关时，LASSO 倾向于只选择其中一个，而弹性网可以同时选择相关特征。

### Dropout

Dropout 在训练时随机将神经元的输出置零：

$$\tilde{h} = h \odot m, \quad m_i \sim \text{Bernoulli}(1 - p)$$

其中 $p$ 是丢弃概率，$\odot$ 是逐元素乘法。想象一个学习小组：每次讨论时随机让一些组员"休息"，剩下的人不能依赖特定同伴，必须自己也学会思考。这迫使神经网络学到更健壮的特征，而不是过度依赖某几个神经元。

测试时，为了保持期望输出不变，需要将权重乘以 $(1 - p)$，或者在训练时使用 Inverted Dropout（将激活值除以 $(1-p)$）。

**Dropout 的理解**：
- 集成学习视角：同时训练 $2^n$ 个子网络（每次随机采样一个）
- 噪声注入视角：向隐藏层添加乘性噪声
- 正则化视角：近似于 L2 正则化和自适应权重衰减

### Batch Normalization

批归一化虽然主要是为了稳定训练，但也具有正则化效果：

$$\hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}$$

其中 $\mu_B$ 和 $\sigma_B^2$ 是批内统计量。

正则化效果来源于：批统计量引入了噪声（取决于当前 batch 中有哪些样本），这种噪声类似于 Dropout 的作用。

## 2.5.6 损失函数的选择指南

| 任务类型 | 推荐损失 | 备注 |
|---------|---------|------|
| 回归 | MSE | 标准选择，假设高斯噪声 |
| 回归（有异常值） | Huber / MAE | 对异常值鲁棒 |
| 二分类 | 交叉熵 | 配合 sigmoid 输出 |
| 多分类 | 交叉熵 | 配合 softmax 输出 |
| 多标签分类 | 二元交叉熵 | 每个标签独立的 sigmoid |
| 不平衡分类 | Focal Loss / 加权交叉熵 | 关注困难样本 |
| 排序 | Hinge / Pairwise Loss | SVM 风格的大间隔损失 |
