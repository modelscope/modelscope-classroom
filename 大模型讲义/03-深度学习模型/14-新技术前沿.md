# 3.14 新技术前沿

深度学习领域发展迅速，新的架构和技术不断涌现。本节简要介绍几个有前景的研究方向：多 Token 预测（MTP）、线性注意力、原生稀疏注意力、状态空间模型（SSM/Mamba），以及扩散生成模型。这些技术代表了突破 Transformer 局限性的不同尝试。

## 3.14.1 多 Token 预测（MTP）

### 标准下一 Token 预测的局限

语言模型的标准训练目标是预测下一个 token：

$$\mathcal{L} = -\sum_{t=1}^T \log P(x_t | x_{<t})$$

每次只预测一个 token，训练信号相对稀疏。而且，模型只"看"一步，可能倾向于局部最优而非全局规划。

### 多 Token 预测

**多 Token 预测**（Multi-Token Prediction, MTP）让模型同时预测未来多个 token：

$$\mathcal{L} = -\sum_{t=1}^T \sum_{k=1}^K \log P(x_{t+k} | x_{<t})$$

模型需要为每个位置输出 $K$ 个预测头。

### 架构实现

简单实现：共享 Transformer 主干，最后用 $K$ 个独立的输出头预测未来 $K$ 个 token。

更复杂的设计可能包含专门的"规划"模块。

### 效果

Meta 的研究表明，MTP 可以：
- 提高样本效率（相同数据量学到更多）
- 改善代码生成等需要规划的任务
- 加速推理（一次生成多个 token）

但 MTP 增加了训练计算量，效果是否值得这些开销仍有争议。

## 3.14.2 线性注意力

### 标准注意力的复杂度

标准 softmax 注意力：

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right) \mathbf{V}$$

计算 $\mathbf{Q}\mathbf{K}^\top \in \mathbb{R}^{n \times n}$ 需要 $O(n^2)$ 时间和空间。

### 线性注意力的思路

**线性注意力**将 softmax 替换为可分解的核函数：

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \frac{\phi(\mathbf{Q})(\phi(\mathbf{K})^\top \mathbf{V})}{\phi(\mathbf{Q})\phi(\mathbf{K})^\top \mathbf{1}}$$

其中 $\phi$ 是特征映射函数。关键观察：先计算 $\phi(\mathbf{K})^\top \mathbf{V} \in \mathbb{R}^{d \times d}$，再与 $\phi(\mathbf{Q})$ 相乘，复杂度变为 $O(nd^2)$。

当 $d \ll n$ 时，这是显著的改进。

### 常见变体

**Linear Transformer**：$\phi(x) = \text{elu}(x) + 1$

**Performer**：使用随机特征近似 softmax 核

**RFA（Random Feature Attention）**：类似 Performer 但有因果性

### 局限性

线性注意力的表达能力通常弱于 softmax 注意力。在语言建模等任务上，性能往往有所下降。

当前线性注意力更多用于需要超长上下文的特定场景，而非通用语言建模。

## 3.14.3 原生稀疏注意力

### 稀疏注意力的分类

**固定稀疏**：预定义哪些位置可以相互注意。

- 滑动窗口：只关注局部 $w$ 个位置
- 块稀疏：将序列分块，只在块内关注
- 跨步稀疏：每隔 $s$ 个位置关注一次

**动态稀疏**：根据内容决定关注哪些位置。

### 硬件高效的稀疏实现

传统稀疏实现在 GPU 上效率不高，因为访存模式不规则。

**Flash Attention** 风格的实现将稀疏模式与 IO 感知算法结合，实现了高效的稀疏注意力。

**Native Sparse Attention**（如 Mistral 的实现）在 Transformer 架构内原生支持稀疏注意力，无需额外的近似或分解。

### Mistral 的滑动窗口

Mistral 使用 $W = 4096$ 的滑动窗口注意力：

- 每个位置只关注前 4096 个位置
- 通过多层堆叠，有效感受野增长
- 复杂度从 $O(n^2)$ 降到 $O(nW)$

配合合理的位置编码（ALiBi 或修改的 RoPE），可以处理超长序列。

## 3.14.4 状态空间模型与 Mamba

### RNN 的复兴？

RNN 被 Transformer 取代的主要原因是无法并行训练。但 RNN 在推理时是 $O(1)$ 的——每个 token 只需常数时间，而 Transformer 需要 $O(n)$（KV cache）或 $O(n^2)$（重新计算）。

能否设计一种既能并行训练、又能高效推理的架构？

### 状态空间模型（SSM）

**状态空间模型**来自控制理论，描述连续时间动态系统：

$$\mathbf{h}'(t) = \mathbf{A}\mathbf{h}(t) + \mathbf{B}x(t)$$
$$y(t) = \mathbf{C}\mathbf{h}(t) + Dx(t)$$

其中 $\mathbf{A}, \mathbf{B}, \mathbf{C}$ 是系统参数，$\mathbf{h}$ 是隐藏状态。

离散化后：

$$\mathbf{h}_k = \bar{\mathbf{A}}\mathbf{h}_{k-1} + \bar{\mathbf{B}}x_k$$
$$y_k = \mathbf{C}\mathbf{h}_k + Dx_k$$

这与 RNN 形式相同，但通过结构化的 $\mathbf{A}$ 矩阵（如对角矩阵）可以高效并行计算。

### S4 模型

**S4**（Structured State Spaces）使用 HiPPO 理论设计 $\mathbf{A}$ 矩阵，在长程依赖任务上表现出色。

S4 的核心技巧是将递归计算转换为卷积，利用 FFT 在 $O(n \log n)$ 时间内计算整个序列。

### Mamba

**Mamba** 在 S4 基础上引入**输入依赖的参数**：

$$\mathbf{B}_t = f_B(x_t), \quad \mathbf{C}_t = f_C(x_t), \quad \Delta_t = f_\Delta(x_t)$$

参数随输入变化，使模型能够"选择性"地记忆或遗忘信息。

Mamba 通过**选择性扫描**（Selective Scan）的硬件高效实现，达到了与 Transformer 相当的训练效率。

### Mamba 的优势与局限

**优势**：
- 推理时复杂度 $O(1)$（固定内存）
- 长序列处理高效
- 在某些任务上媲美 Transformer

**局限**：
- 表达能力可能不如全注意力
- 需要专门的硬件优化内核
- 生态不如 Transformer 成熟

## 3.14.5 扩散生成模型

### 扩散的基本原理

**扩散模型**（Diffusion Model）通过逐步添加和去除噪声来生成数据。

**前向过程**：逐步向数据添加高斯噪声

$$q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t}x_{t-1}, \beta_t\mathbf{I})$$

经过足够多步后，$x_T$ 变成纯噪声。

**逆向过程**：学习从噪声恢复数据

$$p_\theta(x_{t-1} | x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \Sigma_\theta(x_t, t))$$

### 去噪分数匹配

训练目标是预测添加的噪声：

$$\mathcal{L} = \mathbb{E}_{t, x_0, \epsilon}\left[\|\epsilon - \epsilon_\theta(x_t, t)\|^2\right]$$

其中 $\epsilon$ 是添加的噪声，$\epsilon_\theta$ 是网络预测的噪声。

### 条件生成

文本条件控制通过 **Classifier-Free Guidance** 实现：

$$\hat{\epsilon}_\theta(x_t, t, c) = \epsilon_\theta(x_t, t, \emptyset) + w \cdot (\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, \emptyset))$$

其中 $c$ 是条件（如文本嵌入），$w$ 是引导强度。

### 与语言模型的结合

扩散模型在图像生成领域取得了巨大成功（Stable Diffusion、DALL-E 3）。与语言模型结合的方式：

- **级联**：语言模型生成布局/描述，扩散模型生成图像
- **交叉注意力**：文本特征通过交叉注意力引导图像去噪
- **统一架构**：将扩散过程嵌入语言模型框架（如 Transfusion）

### 离散扩散

传统扩散用于连续数据（图像像素）。**离散扩散**将其扩展到离散数据（文本 token）：

- 用掩码代替高斯噪声
- 逐步"揭示"被掩码的 token

这提供了一种非自回归的文本生成方式，可能实现更快的并行生成。

## 3.14.6 展望

这些新技术代表了不同的研究方向：

| 技术 | 目标问题 | 成熟度 |
|------|----------|--------|
| MTP | 训练效率、规划能力 | 初步验证 |
| 线性注意力 | 长序列复杂度 | 理论有限 |
| 稀疏注意力 | 长序列复杂度 | 广泛应用 |
| SSM/Mamba | 推理效率、长序列 | 快速发展 |
| 扩散模型 | 生成质量 | 图像领域成熟 |

Transformer 仍是当前的主流架构，但这些替代方案正在缩小差距。未来可能出现混合架构——结合 Transformer 的表达能力与新技术的效率优势。

对于实践者而言，保持对新技术的关注是必要的，但 Transformer 的深入理解仍是基础。大多数新架构都是在 Transformer 的基础上改进或借鉴其设计思想。
