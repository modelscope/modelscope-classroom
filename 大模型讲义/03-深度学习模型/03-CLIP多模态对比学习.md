# 3.3 CLIP：多模态对比学习

CLIP（Contrastive Language-Image Pre-training）由 OpenAI 于 2021 年提出，是多模态学习的里程碑式工作。它通过对比学习将图像和文本映射到共享的语义空间，使得图像和文本可以直接比较相似度。CLIP 的思想深刻影响了后续的多模态大模型设计，是理解现代视觉-语言模型的重要基础。

## 3.3.1 多模态学习的挑战

### 模态鸿沟

图像和文本是两种本质不同的数据形式。图像是连续的、空间结构化的像素阵列；文本是离散的、序列化的符号。如何让模型“理解”一张图片描述的是“一只猫在草地上奔跑”？这个问题的难度，好比让一个只学过盲文的人和一个只见过照片的人在同一个房间里交流——他们的“语言”完全不同，需要找到一种共同的翻译方式。

传统方法通常是：

1. 为图像标注类别标签（ImageNet 的 1000 类）
2. 训练分类器将图像映射到标签
3. 标签作为图像的"语义表示"

这种方法的局限显而易见：标签空间是预定义的、封闭的。模型只能识别训练时见过的类别，无法处理"戴红色帽子的猫"这种组合性描述。

### 对比学习的思路

CLIP 的核心洞察是：与其将图像映射到固定的标签空间，不如将图像和文本映射到同一个语义空间，让它们可以直接比较。想象一下一场国际商品展览会：与其给每件商品贴上“电子产品”“家居用品”这样固定的标签，不如给每件商品和每段产品描述各分配一个“GPS坐标”，使得相关的商品和描述自然地聚集在坐标空间的同一区域。

训练时，给定一批图像-文本对 $(I_i, T_i)$，目标是：
- 匹配的图像-文本对（如 $I_i$ 和 $T_i$）在语义空间中距离近
- 不匹配的对（如 $I_i$ 和 $T_j$，$i \neq j$）距离远

这就是**对比学习**（Contrastive Learning）的核心思想。

## 3.3.2 CLIP 架构

CLIP 由两个独立的编码器组成：

### 图像编码器

将图像 $I$ 编码为向量 $\mathbf{v} \in \mathbb{R}^d$：

$$\mathbf{v} = f_{\text{image}}(I)$$

CLIP 提供两种图像编码器选项：

**ResNet 变体**：标准的卷积网络，输出全局平均池化后的特征。

**Vision Transformer（ViT）**：将图像分割为 $16 \times 16$ 的 patch，每个 patch 线性投影后作为序列输入 Transformer。使用特殊的 `[CLS]` token 的输出作为图像表示。

ViT 在大规模数据上表现更好，已成为主流选择。

### 文本编码器

将文本 $T$ 编码为向量 $\mathbf{u} \in \mathbb{R}^d$：

$$\mathbf{u} = f_{\text{text}}(T)$$

CLIP 使用 Transformer 编码器，结构类似 GPT-2。文本经过 BPE 分词后，取序列末尾（`[EOS]` token 位置）的隐藏状态作为文本表示。

### 投影到共享空间

两个编码器输出的维度可能不同。CLIP 使用线性投影将它们映射到相同维度的共享空间：

$$\mathbf{v}' = \mathbf{W}_v \mathbf{v}, \quad \mathbf{u}' = \mathbf{W}_u \mathbf{u}$$

投影后的向量经过 L2 归一化：

$$\hat{\mathbf{v}} = \frac{\mathbf{v}'}{\|\mathbf{v}'\|_2}, \quad \hat{\mathbf{u}} = \frac{\mathbf{u}'}{\|\mathbf{u}'\|_2}$$

归一化后，向量的点积等于余弦相似度：

$$\text{sim}(I, T) = \hat{\mathbf{v}}^\top \hat{\mathbf{u}} = \cos(\mathbf{v}', \mathbf{u}')$$

## 3.3.3 对比学习损失

### InfoNCE 损失

给定一批 $N$ 个图像-文本对 $\{(I_1, T_1), \ldots, (I_N, T_N)\}$，CLIP 使用对称的 InfoNCE 损失。

**图像到文本**方向：对于图像 $I_i$，在所有文本中找到匹配的 $T_i$：

$$\mathcal{L}_{I \to T}^{(i)} = -\log \frac{\exp(\hat{\mathbf{v}}_i^\top \hat{\mathbf{u}}_i / \tau)}{\sum_{j=1}^N \exp(\hat{\mathbf{v}}_i^\top \hat{\mathbf{u}}_j / \tau)}$$

**文本到图像**方向：对于文本 $T_i$，在所有图像中找到匹配的 $I_i$：

$$\mathcal{L}_{T \to I}^{(i)} = -\log \frac{\exp(\hat{\mathbf{u}}_i^\top \hat{\mathbf{v}}_i / \tau)}{\sum_{j=1}^N \exp(\hat{\mathbf{u}}_i^\top \hat{\mathbf{v}}_j / \tau)}$$

**总损失**：

$$\mathcal{L} = \frac{1}{2N} \sum_{i=1}^N \left( \mathcal{L}_{I \to T}^{(i)} + \mathcal{L}_{T \to I}^{(i)} \right)$$

### 温度参数

$\tau$ 是可学习的温度参数（temperature），控制相似度分布的"锐度"：

- $\tau$ 小：分布更尖锐，模型更"自信"，对困难负样本更敏感
- $\tau$ 大：分布更平滑，模型更"保守"

CLIP 初始化 $\tau = 0.07$，训练中 $\tau$ 会自适应调整。

### 批次大小的重要性

InfoNCE 的一个特点是：批次越大，负样本越多，对比学习越有效。CLIP 训练使用 32,768 的超大批次。

直觉理解：如果批次只有 2 个样本，区分 $I_1$ 和 $T_1$ 匹配（而非 $I_1$ 和 $T_2$）太容易了——这就像只有两个人的相亲现场，随便配都有 50% 的成功率。当批次有 32,768 个样本时，在 32,768 个候选文本中找到正确的那一个，需要模型真正理解图像内容——这就像在万人相亲大会上准确找到那个对的人，需要真正的“鉴赏力”。

## 3.3.4 训练数据与规模

### WebImageText 数据集

CLIP 的成功很大程度上归功于训练数据。OpenAI 构建了一个包含 4 亿图像-文本对的数据集，称为 WebImageText（WIT）。数据来源是互联网上图像的 alt-text 属性。

与 ImageNet 的 128 万张图片、1000 个类别相比，WIT 的规模和多样性是数量级的提升。更重要的是，文本描述是自然语言，而非预定义的类别标签。

### 训练配置

CLIP 最大的模型（ViT-L/14）训练配置：

- 图像编码器：ViT-L/14（307M 参数）
- 文本编码器：63M 参数
- 批次大小：32,768
- 训练步数：约 12B 图像-文本对
- 32 个 V100 GPU，训练约 12 天

### 规模效应

CLIP 论文展示了清晰的规模效应：更大的模型、更多的数据、更长的训练，性能持续提升。这与后来大语言模型展现的 scaling law 一脉相承。

## 3.3.5 零样本分类

CLIP 最令人印象深刻的能力是**零样本分类**（Zero-shot Classification）：无需任何额外训练，直接在新任务上进行分类。

### 方法

给定一个分类任务（如 ImageNet），将每个类别名转换为文本提示：

```
"a photo of a {class_name}"
```

例如："a photo of a dog"、"a photo of a cat"。

对于待分类的图像 $I$，计算它与所有类别文本的相似度，选择最高的作为预测：

$$\hat{y} = \arg\max_k \text{sim}(I, T_k)$$

### 零样本 ImageNet 结果

CLIP ViT-L/14 在 ImageNet 上的零样本准确率达到 76.2%，与有监督训练的 ResNet-50 相当。这意味着 CLIP 从未见过 ImageNet 的训练数据，却能与专门在该数据集上训练的模型媲美。

### 提示工程

文本提示的设计对性能影响显著。"a photo of a {class_name}" 比简单的 "{class_name}" 效果更好。CLIP 论文尝试了 80 种不同的提示模板，取平均可以进一步提升性能。

这种对提示敏感的特性，与后来大语言模型的 prompt engineering 如出一辙。

## 3.3.6 CLIP 的局限与改进

### 细粒度理解的不足

CLIP 擅长粗粒度的语义匹配，但对细节敏感度不足。例如：

- "红色的苹果"与"绿色的苹果"难以区分
- 空间关系（"猫在桌子上"vs"猫在桌子下"）理解较弱
- 计数能力有限（"三只猫"vs"五只猫"）

### 对比学习的固有限制

对比学习将图像/文本压缩为固定维度的向量。这种“整体表示”丢失了结构信息。图像中的哪部分对应文本中的哪个词？CLIP 无法回答这类问题。这好比你让一个朋友用一个词概括一部电影的全部内容——他可能说“励志”，但你无法从这个词知道主角是谁、结局如何。

### 改进方向

**区域级对比**：如 RegionCLIP，在图像区域与文本短语之间建立对应。

**生成式目标**：如 BLIP，除了对比学习，还训练模型生成图像描述（image captioning）。

**更大规模**：如 EVA-CLIP，使用更大的模型和更多数据，在多个基准上刷新纪录。

## 3.3.7 CLIP 对后续工作的影响

CLIP 的思想被广泛应用于后续的多模态模型：

**DALL-E / Stable Diffusion**：文本到图像生成模型。CLIP 用于引导生成过程，确保生成图像与文本描述一致。

**LLaVA / InternVL**：视觉-语言大模型。使用 CLIP 的视觉编码器提取图像特征，然后与语言模型结合。

**CLIP-based 检索**：图像-文本检索系统。给定文本查询图像，或给定图像查询相关文本。

CLIP 证明了一个重要观点：通过大规模的图像-文本对比学习，可以获得强大的、可迁移的视觉表示。这种表示不依赖于特定任务的标注，具有开放世界的泛化能力。

## 3.3.8 对比学习的数学视角

从信息论的角度，InfoNCE 损失与互信息最大化相关。

### 互信息

两个随机变量 $X, Y$ 的互信息定义为：

$$I(X; Y) = \mathbb{E}_{p(x,y)} \left[ \log \frac{p(x,y)}{p(x)p(y)} \right]$$

互信息度量了 $X$ 和 $Y$ 共享的信息量。如果 $X, Y$ 独立，$I(X; Y) = 0$。

### InfoNCE 作为互信息下界

Oord 等人（2018）证明，InfoNCE 损失提供了互信息的一个下界：

$$I(X; Y) \geq \log N - \mathcal{L}_{\text{InfoNCE}}$$

其中 $N$ 是批次大小。最小化 InfoNCE 损失等价于最大化这个下界。

这解释了为什么批次大小很重要：$N$ 越大，下界越紧，对互信息的估计越准确。

### 表示学习的目标

从这个角度看，CLIP 的目标是学习图像表示 $\mathbf{v}$ 和文本表示 $\mathbf{u}$，使得它们之间的互信息最大化。直觉上，好的表示应该保留两种模态之间共享的语义信息，同时丢弃各自独有的噪声（如图像的像素噪声、文本的表达变体）。
