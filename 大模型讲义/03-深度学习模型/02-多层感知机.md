# 3.2 多层感知机（MLP）

多层感知机（Multi-Layer Perceptron, MLP）是神经网络最基本的组成单元，也是 Transformer 架构中不可或缺的模块。在现代大语言模型中，MLP 承担着特征变换和信息整合的核心角色，其参数量通常占模型总参数的三分之二以上。如果把 Transformer 比作一家公司，注意力层是“会议室”——负责各部门沟通协调，而 MLP 则是每个部门内部的“工作台”——真正处理业务、存储知识的地方。

## 3.2.1 从感知机到多层感知机

### 单层感知机

感知机（Perceptron）由 Rosenblatt 于 1958 年提出，是最简单的线性分类器：

$$y = \text{sign}(\mathbf{w}^\top \mathbf{x} + b)$$

其中 $\text{sign}(z) = 1$ 若 $z > 0$，否则为 $-1$。感知机学习算法在数据线性可分时保证收敛。

然而，Minsky 和 Papert 在 1969 年证明了感知机无法学习 XOR 函数——这个函数不是线性可分的。两个输入 $(0,0), (1,1)$ 输出 0，$(0,1), (1,0)$ 输出 1，没有任何直线能将它们分开。想象一下你是一个门卫，要求“两个人中恰好只来了一个时放行，两个都来或都不来则拦下”。你不可能用一条简单的规则（一条线）来完成这个任务。

### 非线性的必要性

引入隐藏层和非线性激活函数可以解决 XOR 问题。考虑一个两层网络：

$$h_1 = \sigma(x_1 + x_2 - 0.5)$$
$$h_2 = \sigma(x_1 + x_2 - 1.5)$$
$$y = h_1 - h_2$$

当 $\sigma$ 为阶跃函数时，这个网络可以精确计算 XOR。两个隐藏神经元分别编码"至少一个输入为 1"和"两个输入都为 1"，输出层计算它们的差。

若没有非线性激活，多层网络退化为单层线性变换：

$$\mathbf{W}_2 (\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1) + \mathbf{b}_2 = (\mathbf{W}_2 \mathbf{W}_1) \mathbf{x} + (\mathbf{W}_2 \mathbf{b}_1 + \mathbf{b}_2) = \mathbf{W}' \mathbf{x} + \mathbf{b}'$$

矩阵乘法的复合仍是矩阵，无法增加表达能力。回到厨房的场景：如果你只允许做“混合”这一种操作（线性变换），那么不管混合多少次，结果还是某种混合物。但如果允许“加热”（非线性变换），食材就能发生质变——生面团变成面包，蛋液变成蛋饼。非线性激活函数正是这种“加热”操作，是打破线性限制的关键。

## 3.2.2 MLP 的数学形式

标准的 $L$ 层 MLP 定义如下。设输入 $\mathbf{x} \in \mathbb{R}^{d_{\text{in}}}$，输出 $\mathbf{y} \in \mathbb{R}^{d_{\text{out}}}$：

$$\mathbf{h}^{(0)} = \mathbf{x}$$
$$\mathbf{h}^{(l)} = \sigma^{(l)}(\mathbf{W}^{(l)} \mathbf{h}^{(l-1)} + \mathbf{b}^{(l)}), \quad l = 1, \ldots, L-1$$
$$\mathbf{y} = \mathbf{W}^{(L)} \mathbf{h}^{(L-1)} + \mathbf{b}^{(L)}$$

其中 $\mathbf{W}^{(l)} \in \mathbb{R}^{d_l \times d_{l-1}}$，$\mathbf{b}^{(l)} \in \mathbb{R}^{d_l}$，$\sigma^{(l)}$ 是第 $l$ 层的激活函数。

### 隐藏层的作用

隐藏层将输入映射到新的特征空间。以二分类为例，若原始特征线性不可分，隐藏层可以将其变换到线性可分的空间：

$$\mathbf{h} = \sigma(\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1)$$

这个变换相当于非线性的特征提取。与传统机器学习中手工设计特征不同，MLP 从数据中自动学习特征变换。

### 深度与宽度的权衡

**宽度**：单层神经元数量。宽度增加提高该层的表达能力，但参数量线性增长。

**深度**：层数。深度增加使网络可以学习更复杂的层次特征，但带来优化困难和梯度消失问题。

经验和理论都表明，适度增加深度比单纯增加宽度更有效。深度网络可以用更少的参数表达同样复杂的函数。假设你正在学习辨认动物：第一层学会识别边缘和纹理（“这里有一条弧线”），第二层把边缘组合成部件（“这是一只耳朵”），第三层把部件组合成整体（“这是一只猫”）。这种层次化的抽象，每一层只需要学习简单的局部模式，而非一步到位地记住所有可能的动物外观。

## 3.2.3 Transformer 中的 FFN

在 Transformer 架构中，MLP 通常被称为前馈网络（Feed-Forward Network, FFN）或位置前馈层（Position-wise FFN）。其标准形式为：

$$\text{FFN}(\mathbf{x}) = \sigma(\mathbf{x} \mathbf{W}_1 + \mathbf{b}_1) \mathbf{W}_2 + \mathbf{b}_2$$

这里 $\mathbf{x} \in \mathbb{R}^{d_{\text{model}}}$，$\mathbf{W}_1 \in \mathbb{R}^{d_{\text{model}} \times d_{\text{ff}}}$，$\mathbf{W}_2 \in \mathbb{R}^{d_{\text{ff}} \times d_{\text{model}}}$。

### 扩展-压缩结构

FFN 的一个关键设计是**扩展-压缩**（expand-compress）结构：

$$d_{\text{model}} \xrightarrow{\mathbf{W}_1} d_{\text{ff}} \xrightarrow{\mathbf{W}_2} d_{\text{model}}$$

原始 Transformer 设置 $d_{\text{ff}} = 4 \times d_{\text{model}}$。例如，$d_{\text{model}} = 768$ 时，$d_{\text{ff}} = 3072$。这意味着先将特征扩展到更高维空间，应用非线性变换，再压缩回原始维度。

这种设计的直觉是：高维空间有更强的表达能力，非线性变换在高维空间更有效。举个例子：一张平面地图上，两条交叉的路线无法避免相交；但在三维空间中，你可以建一座立交桥让它们各走各的。FFN 的扩展层正扯当这个“升维”角色——把问题搬到更高维的空间里解决，然后再压缩回原始维度。

### 参数量占比

在标准 Transformer 中，FFN 的参数量为：

$$\text{FFN params} = 2 \times d_{\text{model}} \times d_{\text{ff}} = 8 d_{\text{model}}^2$$

而自注意力层的参数量为：

$$\text{Attention params} = 4 \times d_{\text{model}}^2 \quad (\text{Q, K, V, O 四个投影矩阵})$$

FFN 参数量是注意力层的两倍。对于 GPT-3 175B 模型，FFN 参数约占总参数的 65%。

### 位置独立性

"Position-wise"意味着 FFN 对序列中每个位置独立应用相同的变换：

$$\text{FFN}(\mathbf{X})_{i,:} = \text{FFN}(\mathbf{x}_i), \quad \forall i$$

位置之间的交互由注意力层完成，FFN 负责在每个位置进行特征变换。这种分工使架构更加模块化。

## 3.2.4 门控 MLP 变体

现代大语言模型广泛采用门控 MLP 结构，最常见的是 **SwiGLU**（GPT-4、LLaMA 等使用）和 **GeGLU**。

### GLU 门控结构

门控线性单元（Gated Linear Unit, GLU）由 Dauphin 等人于 2017 年提出：

$$\text{GLU}(\mathbf{x}) = (\mathbf{x} \mathbf{W}_1 + \mathbf{b}_1) \otimes \sigma(\mathbf{x} \mathbf{W}_2 + \mathbf{b}_2)$$

其中 $\otimes$ 表示逐元素乘法，$\sigma$ 是 sigmoid 函数。输入被分成两路：一路作为内容，一路作为门控信号。门控信号决定了内容的哪些部分被"通过"。

### SwiGLU

SwiGLU 将 GLU 中的 sigmoid 替换为 Swish 激活函数：

$$\text{SwiGLU}(\mathbf{x}) = (\mathbf{x} \mathbf{W}_1) \otimes \text{Swish}(\mathbf{x} \mathbf{W}_2)$$

其中 $\text{Swish}(z) = z \cdot \sigma(z)$。

为了保持与标准 FFN 相近的参数量，SwiGLU 通常将隐藏维度调整为 $d_{\text{ff}} = \frac{8}{3} d_{\text{model}}$（而非标准的 $4 \times d_{\text{model}}$），因为门控结构多了一个投影矩阵。

LLaMA 系列模型的 FFN 实现：

```python
class LlamaMLP(nn.Module):
    def __init__(self, hidden_size, intermediate_size):
        super().__init__()
        self.gate_proj = nn.Linear(hidden_size, intermediate_size, bias=False)
        self.up_proj = nn.Linear(hidden_size, intermediate_size, bias=False)
        self.down_proj = nn.Linear(intermediate_size, hidden_size, bias=False)
        self.act_fn = nn.SiLU()  # Swish
    
    def forward(self, x):
        return self.down_proj(self.act_fn(self.gate_proj(x)) * self.up_proj(x))
```

注意这里有三个投影矩阵：`gate_proj`、`up_proj`、`down_proj`。

### 门控的效果

门控机制允许网络动态地选择性激活特征。这就像一个经验丰富的厨师：假设你正在做川菜，花椒和豆瓣酱这两种调料都有，但厨师会根据具体菜品决定哪个“通过”（激活）、哪个“关闭”。做麻婆豆腐时两者都激活，做清炒时则都关闭。这提供了更灵活的计算路径，有助于建模复杂的条件依赖。

实验表明，SwiGLU 在相同参数量下优于标准 FFN，已成为大语言模型的默认选择。

## 3.2.5 MLP 的可解释性

近年来，对大语言模型可解释性的研究揭示了 FFN 的一些有趣性质。

### 知识神经元

Dai 等人（2022）发现，FFN 中存在**知识神经元**（Knowledge Neurons）：特定的神经元与特定的事实知识相关联。例如，某些神经元在处理"埃菲尔铁塔位于巴黎"这类事实时会强烈激活。抑制这些神经元会导致模型"遗忘"相关知识。

这表明 FFN 不仅仅是特征变换器，还是知识存储器。注意力层负责信息的路由和组合，FFN 负责存储和检索知识。

### 键值记忆假说

Geva 等人（2021）提出，FFN 的第一层权重 $\mathbf{W}_1$ 可以看作**键**（keys），第二层权重 $\mathbf{W}_2$ 的列向量可以看作**值**（values）。输入与键的匹配程度决定了哪些值被检索出来：

$$\text{FFN}(\mathbf{x}) \approx \sum_i \text{match}(\mathbf{x}, \mathbf{k}_i) \cdot \mathbf{v}_i$$

这种键值记忆的视角与注意力机制有相似之处，但 FFN 是在固定的、参数化的键值对上操作，而注意力在输入依赖的动态键值上操作。

### 稀疏激活

研究发现，FFN 中大量神经元在大多数输入上处于非激活状态（ReLU 输出为 0 或接近 0）。这种**稀疏激活**特性启发了混合专家（MoE）架构：既然大部分神经元在给定输入上不活跃，为何不动态选择一小部分"专家"来处理每个输入？MoE 将在后续章节详细讨论。

## 3.2.6 实现细节

### 偏置项的取舍

现代大语言模型普遍不使用偏置项（bias）。LLaMA、GPT-4 等模型的 FFN 都设置 `bias=False`。理由：

1. 减少参数量（虽然占比很小）
2. 配合 RMSNorm 使用时，偏置项作用有限
3. 简化并行计算

### 初始化策略

FFN 权重通常采用标准正态分布的缩放初始化：

$$\mathbf{W} \sim \mathcal{N}\left(0, \frac{1}{\sqrt{d_{\text{in}}}}\right)$$

这确保了初始输出的方差与输入相当，避免梯度爆炸或消失。

对于残差分支上的最后一层（如 FFN 的输出投影），GPT-2 等模型采用额外的 $\frac{1}{\sqrt{N}}$ 缩放，其中 $N$ 是残差层数。这使得深层网络在初始化时更接近恒等映射。

### 计算效率

FFN 的计算是矩阵乘法，GPU 对此高度优化。对于形状 $(B, L, D)$ 的输入和 $(D, D')$ 的权重：

- FLOPs：$2 \times B \times L \times D \times D'$
- 内存访问：$B \times L \times D + D \times D' + B \times L \times D'$

当 $B \times L$ 较大时（长序列、大批次），FFN 是计算密集型操作，GPU 利用率高。当 $B \times L$ 较小时（单条短文本推理），FFN 变成内存密集型操作，需要频繁加载权重。
