# 3.7 位置编码

Transformer 的自注意力机制对输入序列的顺序是无感知的——对于 `[A, B, C]` 和 `[C, B, A]`，如果没有位置信息，注意力计算的结果完全相同。但语言是有顺序的，"狗咬人"和"人咬狗"意思完全不同。**位置编码**（Positional Encoding）为 Transformer 注入位置信息，是架构设计中的重要一环。

## 3.7.1 位置编码的必要性

### 注意力的置换等变性

考虑自注意力的计算：

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right) \mathbf{V}$$

如果输入序列的顺序改变（施加置换矩阵 $\mathbf{P}$），输出也相应置换：

$$\text{Attention}(\mathbf{P}\mathbf{Q}, \mathbf{P}\mathbf{K}, \mathbf{P}\mathbf{V}) = \mathbf{P} \cdot \text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V})$$

这是**置换等变性**（Permutation Equivariance）。它意味着注意力本身不区分"谁在前谁在后"。

### RNN 的隐式位置编码

RNN 通过顺序处理自然获得位置信息：$\mathbf{h}_t$ 依赖于 $\mathbf{h}_{t-1}$，而 $\mathbf{h}_{t-1}$ 依赖于 $\mathbf{h}_{t-2}$，以此类推。位置信息隐式编码在递归结构中。

Transformer 抛弃了递归，必须显式注入位置信息。

## 3.7.2 正弦位置编码

原始 Transformer 使用**正弦位置编码**（Sinusoidal Positional Encoding）。

### 编码公式

对于位置 $pos$ 和维度 $i$：

$$PE_{pos, 2i} = \sin\left(\frac{pos}{10000^{2i/d}}\right)$$
$$PE_{pos, 2i+1} = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$

其中 $d$ 是嵌入维度，$i \in \{0, 1, \ldots, d/2 - 1\}$。

### 设计直觉

每个维度对应不同"频率"的正弦波：

- 低维度（小 $i$）：高频振荡，编码细粒度位置差异
- 高维度（大 $i$）：低频振荡，编码粗粒度位置关系

这类似于傅里叶级数：用不同频率的正弦波组合表示任意函数。

### 相对位置的线性可表示性

正弦编码的一个关键性质：位置 $pos + k$ 的编码可以表示为位置 $pos$ 编码的线性变换：

$$PE_{pos+k} = f_k(PE_{pos})$$

其中 $f_k$ 是一个只依赖于偏移 $k$ 的线性变换。这由三角恒等式保证：

$$\sin(a + b) = \sin(a)\cos(b) + \cos(a)\sin(b)$$

这意味着模型可以通过学习"关注相邻位置"这类相对关系，而非死记硬背每个绝对位置。

### 实现

```python
def sinusoidal_positional_encoding(max_len, d_model):
    pe = torch.zeros(max_len, d_model)
    position = torch.arange(0, max_len).unsqueeze(1).float()
    div_term = torch.exp(torch.arange(0, d_model, 2).float() * 
                        (-math.log(10000.0) / d_model))
    pe[:, 0::2] = torch.sin(position * div_term)
    pe[:, 1::2] = torch.cos(position * div_term)
    return pe
```

## 3.7.3 可学习位置编码

另一种方法是将位置编码作为可学习参数。

### 绝对位置嵌入

定义位置嵌入矩阵 $\mathbf{P} \in \mathbb{R}^{L_{\max} \times d}$，其中 $L_{\max}$ 是支持的最大序列长度。位置 $i$ 的编码为 $\mathbf{P}[i]$。

训练时，$\mathbf{P}$ 与其他参数一起通过梯度下降优化。

### 与正弦编码的比较

| 特性 | 正弦编码 | 可学习编码 |
|------|----------|------------|
| 参数量 | 0 | $L_{\max} \times d$ |
| 泛化性 | 可外推（理论上） | 长度外推困难 |
| 灵活性 | 固定模式 | 数据驱动 |

实验表明，两者性能相当。BERT、GPT-2 使用可学习位置编码；原始 Transformer 使用正弦编码。

### 外推问题

可学习位置编码的一个严重问题是**长度外推**（Length Extrapolation）：如果训练时最大长度是 2048，模型无法处理长度为 4096 的输入，因为位置 2048-4095 的嵌入从未被训练过。

正弦编码理论上可以外推，但实践中注意力模式也会因位置分布变化而失效。

## 3.7.4 旋转位置编码（RoPE）

**旋转位置编码**（Rotary Position Embedding, RoPE）由苏剑林（2021）提出，已成为现代大语言模型的主流选择（LLaMA、Qwen、Mistral 等）。

### 核心思想

RoPE 不是将位置编码加到嵌入上，而是在计算注意力时，对 Query 和 Key 向量施加与位置相关的旋转。

设 $\mathbf{q}_m$ 是位置 $m$ 的 Query 向量，$\mathbf{k}_n$ 是位置 $n$ 的 Key 向量。RoPE 将它们变换为：

$$\tilde{\mathbf{q}}_m = R_m \mathbf{q}_m, \quad \tilde{\mathbf{k}}_n = R_n \mathbf{k}_n$$

其中 $R_m$ 是一个旋转矩阵。

### 相对位置的自然编码

注意力分数变为：

$$\tilde{\mathbf{q}}_m^\top \tilde{\mathbf{k}}_n = \mathbf{q}_m^\top R_m^\top R_n \mathbf{k}_n = \mathbf{q}_m^\top R_{n-m} \mathbf{k}_n$$

注意 $R_m^\top R_n = R_{n-m}$（旋转矩阵的性质）。这意味着注意力分数只依赖于**相对位置** $n - m$，而非绝对位置 $m$ 和 $n$。

### 二维旋转的实现

对于二维向量 $[x, y]$，旋转角度 $\theta$ 的变换是：

$$R_\theta \begin{bmatrix} x \\ y \end{bmatrix} = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix} \begin{bmatrix} x \\ y \end{bmatrix}$$

RoPE 将 $d$ 维向量分成 $d/2$ 对，每对独立应用不同频率的旋转：

$$\theta_i = 10000^{-2i/d}$$

位置 $m$ 的旋转角度为 $m \cdot \theta_i$。

### 高效实现

旋转可以用复数乘法高效实现。将 $[x_{2i}, x_{2i+1}]$ 视为复数 $x_{2i} + i \cdot x_{2i+1}$，旋转变为：

$$(x_{2i} + i \cdot x_{2i+1}) \cdot e^{i m \theta_i} = (x_{2i} + i \cdot x_{2i+1}) \cdot (\cos(m\theta_i) + i \sin(m\theta_i))$$

代码实现：

```python
def apply_rope(x, cos, sin):
    # x: (batch, seq_len, n_heads, head_dim)
    x1, x2 = x[..., ::2], x[..., 1::2]
    return torch.cat([x1 * cos - x2 * sin, x1 * sin + x2 * cos], dim=-1)
```

### RoPE 的优势

1. **相对位置编码**：注意力天然基于相对位置
2. **长度外推性**：理论上可处理任意长度
3. **计算高效**：只需逐元素乘法
4. **无额外参数**：频率是预设的，不需要学习

## 3.7.5 ALiBi：无需位置编码

**ALiBi**（Attention with Linear Biases）由 Press 等人（2022）提出，采用完全不同的思路：不修改嵌入，而是在注意力分数上加偏置。

### 方法

在注意力分数计算时，加入与距离成比例的惩罚：

$$\text{Attention}_{i,j} = \text{softmax}\left(\frac{\mathbf{q}_i^\top \mathbf{k}_j}{\sqrt{d_k}} - m \cdot |i - j|\right)$$

其中 $m$ 是一个预设的斜率（不同头使用不同斜率）。距离越远的位置，注意力分数受到的惩罚越大。

### 外推能力

ALiBi 的一个显著优点是优秀的**长度外推**能力。由于偏置只依赖于相对距离，模型可以自然推广到更长的序列。

实验表明，在 1024 长度上训练的 ALiBi 模型可以直接在 2048 甚至更长的序列上使用，性能下降很小。

### 与 RoPE 的比较

| 特性 | RoPE | ALiBi |
|------|------|-------|
| 修改位置 | Q, K 向量 | 注意力分数 |
| 位置类型 | 相对位置（旋转） | 相对位置（线性偏置） |
| 外推能力 | 中等 | 强 |
| 表达能力 | 更强（非线性） | 较弱（线性） |

## 3.7.6 长度外推与内插

### 外推问题

所有位置编码方法在处理训练时未见过的长度时都会面临挑战。原因：

1. 注意力模式在长序列上发生变化
2. 位置编码的数值范围超出训练分布

### YaRN：RoPE 的外推改进

YaRN（Yet another RoPE extensioN）通过修改 RoPE 的频率来改善外推：

$$\theta'_i = \theta_i / s$$

其中 $s$ 是缩放因子。直觉上，将频率降低使得旋转角度增长更慢，从而能够覆盖更长的位置。

### NTK-Aware Interpolation

另一种方法是在高频和低频上使用不同的缩放策略：

- 低频分量（编码粗粒度位置）：较大缩放
- 高频分量（编码细粒度位置）：较小缩放

这基于神经正切核（Neural Tangent Kernel）的理论分析。

### 实践建议

1. 如果训练长度足够（如 8K），RoPE 在 2x 长度内外推效果尚可
2. 需要更长外推时，考虑 YaRN 或重新训练
3. ALiBi 天然具有更好的外推能力，但可能牺牲一些性能
4. 最可靠的方法是在目标长度上进行继续训练

## 3.7.7 位置编码的选择

### 主流模型的选择

| 模型 | 位置编码 | 训练长度 |
|------|----------|----------|
| GPT-2 | 可学习绝对位置 | 1024 |
| BERT | 可学习绝对位置 | 512 |
| GPT-3 | 可学习绝对位置 | 2048 |
| LLaMA | RoPE | 2048-4096 |
| LLaMA 2 | RoPE + 扩展 | 4096 |
| Qwen | RoPE | 8192+ |
| Mistral | RoPE + Sliding Window | 8192 |
| BLOOM | ALiBi | 2048 |

### 设计考量

1. **训练长度**：决定了不使用外推技术时的最大可用长度
2. **外推需求**：如果应用需要处理比训练更长的序列，优先考虑 RoPE 或 ALiBi
3. **计算效率**：RoPE 略增加计算量（约 5%），ALiBi 几乎无额外开销
4. **兼容性**：不同位置编码不兼容，迁移预训练权重时需注意

## 3.7.8 位置编码与注意力模式

位置编码深刻影响注意力的学习模式。

### 局部偏好

正弦编码和 RoPE 都倾向于让模型学习"关注附近位置"的模式。这与语言的局部依赖性一致：理解一个词通常需要看它的上下文，而非遥远的位置。

### 长程依赖

对于需要长程依赖的任务（如长文档问答），位置编码的外推能力至关重要。如果模型在位置 100 和位置 3000 之间的注意力权重过低，它将无法捕获跨越长距离的关系。

### 可视化分析

分析注意力模式时，常常可以看到：

- 早期层：关注局部上下文
- 中间层：开始形成更广的注意力范围
- 后期层：注意力模式更稀疏、更任务相关

位置编码的选择影响这些模式的形成。RoPE 倾向于产生更平滑的衰减，ALiBi 产生更明确的局部偏好。
